# Preview parity

The files in `out/verify/previews` are generated by `RasterPreviewGenerator` and
`PreviewGenerator`. No generated design image is used as an input or copied into them.

| Capability | AWT/SVG preview | Minecraft fallback | Enhanced OpenGL path |
| --- | --- | --- | --- |
| Scene/model/layout | Same production Java code | Same production Java code | Same recorded commands |
| Input and settings | Covered by shared tests | Routed by endpoint adapter | Same scene behavior |
| Colors/radii/clips | Exact semantic values | Rounded native approximation | SDF target |
| Glass | CPU/SVG approximation | Rounded translucent tint | Shared Kawase blur plan |
| Font | System Sans/Mono metrics | Scaled Minecraft font | Client-supplied atlas |
| Background | Deterministic block-world fixture | Live game framebuffer | Live captured framebuffer |

Therefore the previews are evidence of implemented layout and drawing behavior, but they
are not a promise that the compatibility renderer has identical font rasterization or
framebuffer blur. A consuming client obtains that last visual parity by binding
`CommandRenderer`/`GlCommandSink` to its GL state, texture and font services.

The reference Myau screenshot is treated only as an information-architecture reference.
OpenMyau-Plus is GPL-3.0 and contains several different ClickGUI implementations; no source
from it is vendored into Kairos.
