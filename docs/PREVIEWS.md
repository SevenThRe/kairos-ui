# Preview parity

ClickGUI PNG/SVG previews were removed together with the native workbench and panel
scenes. They are not used as a proxy for the MCEF result. The production ClickGUI source
is the offline Web bundle under
`integrations/liquidbounce-1.12.2/overlay/shared/main/resources/assets/kairos_ui/web`.
It receives the real LiquidBounce module catalog through `KairosWebBridge`; the Forge
screen supplies input, a transparent Chromium surface, and the live blurred framebuffer.

The files still emitted under `out/verify/previews` are generated by
`RasterPreviewGenerator` and `PreviewGenerator` for native HUD and ESP code only.

| Capability | Headless HUD/ESP preview | Minecraft runtime |
| --- | --- | --- |
| Scene/model/layout | Same native Java code | Same native Java code |
| Colors/radii/clips | Exact semantic values | Endpoint renderer |
| Entity/item | Intentionally blank native slots | Native entity/ItemStack hook |
| Background | Deterministic block-world fixture | Live framebuffer |

`combat-hud.png` and `competitive-esp.png` are generated in the same verification run.
They demonstrate the compact TargetHUD, detailed inspector, equipment/durability slots,
outlined player boxes, health bars, held-item tags, armor values, and bed/chest/item boxes.

Therefore these previews are evidence only for HUD/ESP layout and drawing behavior. The
ClickGUI is verified as a bundled Web application plus an origin-checked runtime bridge.
GitHub Actions then compiles it into the combined LiquidBounce/MCEF Forge artifact and
inspects the resulting JAR.

The reference Myau screenshot is treated only as an information-architecture reference.
OpenMyau-Plus is GPL-3.0 and contains several different ClickGUI implementations; no source
from it is vendored into Kairos.
